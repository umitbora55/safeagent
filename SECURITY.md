# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SafeAgent, please report it responsibly.

**Email:** safeagent.security@proton.me
**GitHub:** [Security Advisories](https://github.com/umitbora55/safeagent/security/advisories/new)

**Do not** open public GitHub issues for security vulnerabilities.

## Vulnerability Disclosure Policy

- Please include a clear description, impact, and reproduction steps.\n- We will acknowledge receipt and keep you updated on remediation.\n+- Public disclosure should wait until a fix is available.\n+
## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Triage:** Within 5 business days
- **Fix target:** Within 7 days for critical issues
- **Disclosure:** Coordinated with reporter

## Scope

- Vault encryption bypass
- Prompt injection guard bypass
- Credential exposure in logs or output
- Dependency vulnerabilities

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x (v1-core) | Yes |
| < 1.0.0 | No |
